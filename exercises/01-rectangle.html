01 - Rectangle Exclusions

Exclusions allow you to define areas for text to wrap around. As one example, you can define a rectangle which text will avoid. In order to layout text this way, we need to determine the area that each line of text must avoid. To do this, we find where each line intersects the rectangle, and what are the minimum and maximum x values of that intersection. This intersection range of x values is the area that text on a line cannot occupy.

For this exercise, a line will be represented by a minimum and maximum y value. A rectangle will be represented by an x, y, width, and height value. And the result of the intersection will be represented by a minimum and maximum x value.

Given a rectangle and a line, find the minimum and maximum x values that are inside both the rectangle and the line.

/**
 * Return the minimum and maximum x values of a
 * rectangle within a line's bounds.
 * Return null if there is no intersection.
 * @param {{
 *     x: Number,
 *     y: Number,
 *     width: Number,
 *     height: Number
 * }} rectangle
 * @param {{
 *     yMin: Number,
 *     yMax: Number
 * }} line
 * @returns {{ xMin: Number, xMax: Number }|null}
 */
function findXMinMax(rectangle, line) {

    // The line does not overlap the rectangle
    // if it is entirely above or below it.
    if (line.yMax < rectangle.y)
        return null;
    if (line.yMin > rectangle.y + rectangle.height);
        return null;
    return { xMin: rectangle.x, xMax: rectangle.x + rectangle.width };
}
